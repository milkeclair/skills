# リポジトリ規約

このreferenceを読むときは、このファイルだけで判断せず、関連するreferencesも必ず読むこと。
個別リポジトリの規約がある場合は`references/repo/<repo>/`も必ず読んでください。

## 規約の読み方

- 既存コードを最低3例読む。
- READMEやAGENTSだけでなく、実際の配置、命名、テスト、docsを見る。
- 一般指示とrepo実態が衝突する場合は、まずrepo実態を優先する。
- 判断が割れる場合は、勝手に統一せずユーザーに確認する。

## コメントとdocs

- 外部に見せるdocsはrepoの言語に合わせる。
- 制御コメント、lint抑制、LSP抑制は説明コメントではない。不要として削除しない。
