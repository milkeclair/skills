# 完了前レビュー

このreferenceを読むときは、このファイルだけで判断せず、関連するreferencesも必ず読むこと。
個別リポジトリの規約がある場合は`references/repo/<repo>/`も必ず読んでください。

## 必須確認

- ユーザー指示に答えているか。
- 最新の質問、対象symbol、対象artifactに答えているか。
- TODOの完了条件ごとに対応する証拠があるか。
- 依頼外変更が混ざっていないか。
- 触った全ファイルを読み返したか。
- docs、help、README、testsが最新仕様に揃っているか。
- 古い名称、古い使い方、古いpathが残っていないか。
- process、Docker、生成ファイルの残りがないか。
- 最終artifactが最後の変更後に作られているか。
- 全件確認を主張する場合、missing、duplicate、issue、uncheckedが残っていないか。
- テスト未実行の場合、その理由が明確か。

## 報告

- 何を変えたかを短く具体的に書く。
- なぜその問題が起きたかを書く。
- 実行した確認と結果を書く。
- できなかった確認を隠さない。
- 推測を事実として書かない。
- 途中経過を完了として書かない。
- corpus全件の実証と形式証明を区別し、証拠より強い表現を使わない。
