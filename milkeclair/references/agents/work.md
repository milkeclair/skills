## ルール

- webへのfetchを利用して、情報を参照すること。
- 参照した情報のリンクを回答に含めること。
- 動作確認は必ず行う。推測での変更は認めない。
  - 動作確認に関して必要な情報が足りない場合は、自力でコードベースを調べる。
- コードから明示的なコメントは書かない。
  - 例:
    ```ruby
    # This method calculates the sum of two numbers
    def sum(a, b)
      a + b
    end
    ```
    上記のコメントはコードを見れば明らかなので不要。
- コメントは基本的に日本語で書く。既存のスタイルが英語であればそれに合わせる。
- 型システムがある言語(例: Typescript)において、asなどの安全でないアサーションを使用しない。
  - anyも禁止。
  - ただし、既存コードで使用されている場合は例外。
  - また、asやanyを使用しないことで非常に複雑な型パズルになる場合は、コメントで理由を説明することで例外的に許可する。
